
[size=20][b]Понимание схем[/b][/size]

В Source 1 сущности имели два типа свойств: [b]send props[/b] (синхронизируемые с клиентами) и [b]data props[/b] (только на сервере). Source 2 упростил это, объединив их в единую систему [b]схем[/b]. Схемы определяют структуру и свойства всех игровых объектов, и многие поля схем автоматически передаются по сети.

Схемы позволяют вам получать доступ и изменять свойства любой сущности или объекта в Source 2, включая здоровье, позицию, скорость, модель, команду и бесчисленное количество других атрибутов. Когда вы изменяете сетевые поля схем с установленным [font=monospace]changeState[/font] в [font=monospace]true[/font], изменения автоматически синхронизируются с клиентами.

[size=20][b]Поиск информации о схемах[/b][/size]

Вы можете найти названия классов схем и имена членов, используя:
[list]
[*][b]Онлайн-дампы[/b]: [url=https://github.com/a2x/cs2-dumper/blob/main/output/server_dll.hpp]репозиторий cs2-dumper[/url]
[*][b]Дамперы схем[/b]: Различные инструменты сообщества для извлечения определений схем
[*][b]Файлы ресурсов игры[/b]: Расположены в папках схем вашей игры
[/list]

Распространенные классы схем включают [font=monospace]CBaseEntity[/font], [font=monospace]CBasePlayerPawn[/font], [font=monospace]CCSPlayerController[/font], [font=monospace]CBaseModelEntity[/font] и многие другие.

[size=20][b]Методы Schema vs методы Data[/b][/size]

s2sdk предоставляет два подхода для работы со схемами:

[size=16][b]Методы Schema (рекомендуется)[/b][/size]
Используют имена классов и членов в виде строк. Они проще в использовании и более читаемы:
[code=csharp]SetEntSchema(entityHandle, "CBaseEntity", "m_iHealth", 200, true, 0);
int health = GetEntSchema(entityHandle, "CBaseEntity", "m_iHealth", 0);[/code]

[size=16][b]Методы Data (расширенные)[/b][/size]
Используют необработанные смещения в памяти. Они быстрее, но требуют ручного управления смещениями:
[code=csharp]int offset = GetSchemaOffset("CBaseEntity", "m_iHealth");
SetEntData(entityHandle, offset, 200, 4, true, 0);
int health = GetEntData(entityHandle, offset, 4);[/code]

[size=20][b]Handle сущностей vs указатели[/b][/size]

Большинство функций поставляются в двух версиях:

[list]
[*][b]Версия с handle[/b] (например, [font=monospace]GetEntSchema[/font]): Работает с handle сущностей - используйте это по возможности
[*][b]Версия с указателем[/b] (например, [font=monospace]GetEntSchema2[/font]): Работает с необработанными указателями - используйте только когда handle не могут быть сгенерированы (некоторые схемы не основаны на сущностях)
[/list]

Всегда предпочитайте handle сущностей, если только вы не работаете с объектами схем, не являющимися сущностями.

[size=20][b]Работа с массивами[/b][/size]

Некоторые поля схем являются массивами. Используйте параметр [font=monospace]element[/font] для доступа к определенным индексам массива:

[code=csharp]// Получение размера массива
int arraySize = GetEntSchemaArraySize(entityHandle, "SomeClass", "m_arrayField");

// Доступ к элементам массива (параметр element - последний перед параметрами по умолчанию для конкретного типа)
for (int i = 0; i < arraySize; i++)
{
    int value = (int)GetEntSchema(entityHandle, "SomeClass", "m_arrayField", i);
    PrintToServer($"Array[{i}] = {value}\n");
}

// Установка элемента массива
SetEntSchema(entityHandle, "SomeClass", "m_arrayField", 42, true, 2); // Установить индекс 2 в 42[/code]

[size=20][b]Понимание параметров[/b][/size]

[size=16][b]changeState (bool)[/b][/size]
Когда [font=monospace]true[/font], помечает поле как измененное и запускает сетевую синхронизацию с клиентами. Устанавливайте в [font=monospace]true[/font] при изменении сетевых полей, [font=monospace]false[/font] для изменений только на сервере.

[size=16][b]element (int)[/b][/size]
Индекс массива при работе с полями массивов схем. Используйте [font=monospace]0[/font] для полей, не являющихся массивами.

[size=16][b]chainOffset (int)[/b][/size]
Расширенный параметр для смещений цепочки в иерархиях наследования. Используйте [font=monospace]0[/font] для большинства случаев, или используйте [font=monospace]GetSchemaChainOffset()[/font] для конкретных цепочек наследования. Используйте [font=monospace]-1[/font] для классов, не являющихся сущностями.

[size=16][b]Пример 1: Переключение режима бога[/b][/size]


[code=csharp]using Plugify;
using static s2sdk.s2sdk;

public unsafe class GodMode : Plugin
{
    private bool[] godMode = new bool[65]; // Max players

    public void OnPluginStart()
    {
        AddConsoleCommand("god", "Toggle god mode",
            ConVarFlag.LinkedConcommand | ConVarFlag.Release,
            Command_God, HookMode.Post);
    }

    public ResultType Command_God(int caller, int context, string[] arguments)
    {
        if (caller == -1) return ResultType.Handled;

        godMode[caller] = !godMode[caller];

        int player = PlayerSlotToEntHandle(caller);

        if (godMode[caller])
        {
            SetEntSchema(player, "CBaseEntity", "m_iHealth", 999, true, 0);
            SetEntSchema(player, "CBaseEntity", "m_takedamage", 0, true, 0); // DAMAGE_NO
            PrintToServer("God mode enabled\n");
        }
        else
        {
            SetEntSchema(player, "CBaseEntity", "m_iHealth", 100, true, 0);
            SetEntSchema(player, "CBaseEntity", "m_takedamage", 2, true, 0); // DAMAGE_YES
            PrintToServer("God mode disabled\n");
        }

        return ResultType.Handled;
    }
}[/code]

[code=csharp]using System.Numerics;
using Plugify;
using static s2sdk.s2sdk;

public unsafe class Teleport : Plugin
{
    public void OnPluginStart()
    {
        AddConsoleCommand("teleport", "Teleport to coordinates",
            ConVarFlag.LinkedConcommand | ConVarFlag.Release,
            Command_Teleport, HookMode.Post);
    }

    public ResultType Command_Teleport(int caller, int context, string[] arguments)
    {
        if (caller == -1) return ResultType.Handled;
        if (arguments.Length < 4)
        {
            PrintToServer("Usage: teleport <x> <y> <z>\n");
            return ResultType.Handled;
        }

        int player = PlayerSlotToEntHandle(caller);

        float x = float.Parse(arguments[1]);
        float y = float.Parse(arguments[2]);
        float z = float.Parse(arguments[3]);

        Vector3 newPosition = new Vector3(x, y, z);
        SetEntSchemaVector3D(player, "CBaseEntity", "m_vecAbsOrigin", newPosition, true, 0);

        // Also reset velocity to prevent momentum
        Vector3 zeroVelocity = new Vector3(0, 0, 0);
        SetEntSchemaVector3D(player, "CBaseEntity", "m_vecAbsVelocity", zeroVelocity, true, 0);

        PrintToServer($"Teleported to ({x}, {y}, {z})\n");

        return ResultType.Handled;
    }
}[/code]

[size=20][b]Расширенное: использование необработанных смещений[/b][/size]

Для критичного по производительности кода вы можете использовать необработанные смещения:

[code=csharp]// Получите смещение один раз и кешируйте его
int healthOffset = GetSchemaOffset("CBaseEntity", "m_iHealth");
int chainOffset = GetSchemaChainOffset("CBaseEntity");

// Используйте смещение напрямую для лучшей производительности
SetEntData(entityHandle, healthOffset, 100, 4, true, chainOffset);
int health = (int)GetEntData(entityHandle, healthOffset, 4);[/code]

[size=20][b]Использование версий с указателями[/b][/size]

Для объектов схем, не являющихся сущностями (когда handle недоступны):

[code=csharp]// Пример: работа с объектом схемы, не являющимся сущностью
nint objectPointer = GetSomeNonEntityPointer();

// Используйте версию функций с '2'
SetEntSchema2(objectPointer, "SomeClass", "m_someField", 42, true, 0);
int value = (int)GetEntSchema2(objectPointer, "SomeClass", "m_someField", 0);[/code]

[size=20][b]Обновления сетевого состояния[/b][/size]

Для ручного запуска сетевых обновлений:

[code=csharp]// Обновить конкретное поле
NetworkStateChanged(entityHandle, "CBaseEntity", "m_iHealth");

// Или используя смещение
int offset = GetSchemaOffset("CBaseEntity", "m_iHealth");
int chainOffset = GetSchemaChainOffset("CBaseEntity");
ChangeEntityState(entityHandle, offset, chainOffset);[/code]

[size=20][b]Советы и лучшие практики[/b][/size]

[list=1]
[*][b]Используйте методы schema вместо методов data[/b] - Они более читаемы и удобны в обслуживании
[*][b]Предпочитайте handle сущностей указателям[/b] - Используйте версии с указателями только при необходимости
[*][b]Кешируйте смещения в критичном по производительности коде[/b] - Избегайте повторных вызовов [font=monospace]GetSchemaOffset()[/font]
[*][b]Всегда устанавливайте changeState правильно[/b] - Используйте [font=monospace]true[/font] для сетевых изменений, [font=monospace]false[/font] только для сервера
[*][b]Обращайтесь к дампам схем[/b] - Проверьте [url=https://github.com/Wend4r/s2d-schema]s2d-schema[/url] для доступных полей
[*][b]Проверяйте handle сущностей[/b] - Убедитесь, что сущности действительны перед доступом к схемам
[*][b]Используйте соответствующие типы данных[/b] - Сопоставляйте функцию с типом поля схемы (Int, Float, String, Vector и т.д.)
[*][b]Тестируйте с клиентами[/b] - Проверьте, что сетевые изменения синхронизируются правильно
[/list]
