
[size=20][b]Понимание User Messages[/b][/size]

User messages - это основной метод отправки пользовательских данных с сервера клиентам в Source 2. Они используют [b]Protocol Buffers (protobuf)[/b] для сериализации, обеспечивая структурированный и эффективный способ передачи игровой информации.

Типичные случаи использования включают:
[list]
[*]Отображение HUD-элементов и уведомлений
[*]Воспроизведение звуков или визуальных эффектов на клиентах
[*]Отправка пользовательской информации о состоянии игры
[*]Создание чат-сообщений и подсказок
[*]Запуск событий на стороне клиента
[/list]

User messages работают аналогично реализации SourceMod, где protobuf заменяет старую систему bitbuffer для лучшей типобезопасности и гибкости.

[quote][b]Info:[/b] [b]Новое[/b]: Класс [font=monospace]UserMessage[/font] обеспечивает автоматическое управление ресурсами и более чистый синтаксис для работы с user messages. Он гарантирует правильную очистку и предоставляет интуитивные методы![/quote]

Каждое определение сообщения показывает имя сообщения, ID и доступные поля с их типами.

using Plugify;
using static s2sdk.s2sdk;

public unsafe class Sample : Plugin
{
    public void OnPluginStart()
    {
        // Найти ID сообщения по имени
        short msgId = UserMessageFindMessageIdByName("TextMsg");

        // Перехватить сообщение в режиме Post
        HookUserMessage(msgId, OnTextMessage, HookMode.Post);
    }

    public void OnPluginEnd()
    {
        short msgId = UserMessageFindMessageIdByName("TextMsg");
        UnhookUserMessage(msgId, OnTextMessage, HookMode.Post);
    }

    private static void OnTextMessage(nint userMessage)
    {
        // Прочитать данные сообщения
        string text = PbReadString(userMessage, "text", 0);
        PrintToServer($"TextMsg intercepted: {text}\n");
    }
}

[size=20][b]Создание и отправка User Messages[/b][/size]

[spoiler=С классами (Рекомендуется)]
Создавайте user messages используя класс [font=monospace]UserMessage[/font] с автоматической очисткой:
[code=csharp]
  // Создать из имени сообщения (наиболее распространенный способ)
  using (var msg = new UserMessage("TextMsg"))
  {
      // Установить получателей
      msg.AddAllPlayers();

      // Установить поля сообщения
      msg.SetString("text", "Hello from server!");
      msg.SetInt32("msg_dst", 4);  // HUD_PRINTTALK

      // Отправить сообщение
      msg.Send();
  } // Автоматически уничтожается

  // Или создать из ID сообщения
  short msgId = UserMessage.FindMessageIdByName("TextMsg");
  using (var msg = new UserMessage(msgId))
  {
      // Настроить и отправить...
  }
  
[/code]
[/spoiler]

[spoiler=Без классов]
Создавайте user messages используя явные функции:
[code=csharp]
  // Метод 1: Из имени сообщения (наиболее распространенный)
  nint msg = UserMessageCreateFromName("TextMsg");

  // Метод 2: Из ID сообщения
  short msgId = UserMessageFindMessageIdByName("TextMsg");
  nint msg = UserMessageCreateFromId(msgId);

  // Метод 3: Из serializable (продвинутый)
  nint msg = UserMessageCreateFromSerializable(msgSerializable, message, recipientMask);

  // Установить получателей (выберите один подход)
  UserMessageAddAllPlayers(msg);  // Все игроки
  // ИЛИ
  UserMessageAddRecipient(msg, playerSlot);  // Один игрок
  // ИЛИ
  UserMessageSetRecipientMask(msg, customMask);  // Пользовательская маска

  // Установить поля сообщения
  PbSetString(msg, "text", "Hello from server!");
  PbSetInt32(msg, "msg_dst", 4);  // HUD_PRINTTALK

  // Отправить сообщение
  UserMessageSend(msg);

  // Очистить (ВАЖНО!)
  UserMessageDestroy(msg);
  
[/code]
[/spoiler]

[size=20][b]Работа с повторяющимися полями[/b][/size]

Некоторые protobuf-поля являются массивами (repeated fields):

[spoiler=С классами (Рекомендуется)]
[code=csharp]
  using (var msg = new UserMessage("CustomMsg"))
  {
      // Добавить значения в повторяющееся поле
      msg.AddInt32("items", 10);
      msg.AddInt32("items", 20);
      msg.AddInt32("items", 30);
      msg.AddString("names", "Player1");
      msg.AddString("names", "Player2");

      // Получить количество элементов в повторяющемся поле
      int count = msg.GetRepeatedFieldCount("items");
      PrintToServer($"Total items: {count}\n");

      // Прочитать повторяющиеся значения
      for (int i = 0; i < count; i++)
      {
          int value = msg.GetInt32("items", i);
          PrintToServer($"Item[{i}] = {value}\n");
      }

      // Установить значение по конкретному индексу в повторяющемся поле
      msg.SetRepeatedInt32("items", 0, 999);  // Установить индекс 0 в 999

      // Удалить значение из повторяющегося поля
      msg.RemoveRepeatedFieldValue("items", 0);  // Удалить индекс 0
  }
  
[/code]
[/spoiler]

[spoiler=Без классов]

[size=20][b]Работа с вложенными сообщениями[/b][/size]

[spoiler=С классами (Рекомендуется)]

  using (var msg = new UserMessage("ComplexMsg"))
  {
      // Получить вложенное сообщение
      var nestedMsg = msg.GetMessage("nested_field");
      if (nestedMsg != null)
      {
          // Прочитать из вложенного сообщения
          int value = nestedMsg.GetInt32("value", 0);
          string name = nestedMsg.GetString("name", 0);

      // Получить повторяющееся вложенное сообщение
      int count = msg.GetRepeatedFieldCount("repeated_nested");

          var repeatedMsg = msg.GetRepeatedMessage("repeated_nested", i);
          if (repeatedMsg != null)

      // Добавить вложенное сообщение в повторяющееся поле
      var newMsg = new UserMessage("PlayerInfo");
      newMsg.SetString("name", "NewPlayer");
      newMsg.SetInt32("score", 100);

[size=20][b]Управление получателями[/b][/size]

Контролируйте, кто получает сообщение:

[spoiler=С классами (Рекомендуется)]
[code=csharp]
  using (var msg = new UserMessage("TextMsg"))
  {
      // Отправить всем игрокам
      msg.AddAllPlayers();

      // ИЛИ отправить конкретному игроку
      msg.AddRecipient(playerSlot);

      // ИЛИ отправить нескольким конкретным игрокам
      msg.AddRecipient(0);  // Слот игрока 0
      msg.AddRecipient(3);  // Слот игрока 3
      msg.AddRecipient(7);  // Слот игрока 7

      // ИЛИ использовать маску получателей (продвинутый)
      ulong mask = 0x1F;  // Бинарная маска для первых 5 игроков
      msg.SetRecipientMask(mask);

      // Получить текущую маску получателей
      ulong currentMask = msg.GetRecipientMask();
      PrintToServer($"Recipient mask: 0x{currentMask:X}\n");

      // Настроить поля сообщения...
      msg.SetString("text", "Hello!");

      // Отправить
      msg.Send();
  }
  
[/code]
[/spoiler]

        // Изменить сообщение
        if (filtered != originalText)
        {
            userMessage.SetString("text", filtered);
            PrintToServer("Filtered message content\n");
        }

        // Добавить префикс ко всем сообщениям
        userMessage.SetString("text", "[Server] " + filtered);

    }

[size=20][b]Советы и лучшие практики[/b][/size]

[list=1]
[*][b]Всегда уничтожайте сообщения[/b] - Вызывайте [font=monospace]UserMessageDestroy()[/font] после отправки, чтобы предотвратить утечки памяти
[*][b]Проверяйте ID сообщений[/b] - Убедитесь, что [font=monospace]UserMessageFindMessageIdByName()[/font] возвращает действительный ID перед использованием
[*][b]Сначала устанавливайте получателей[/b] - Настраивайте получателей перед установкой значений полей
[*][b]Используйте правильные имена полей[/b] - Обращайтесь к определениям protobuf для точных имен полей
[*][b]Правильно обрабатывайте режимы перехвата[/b] - Используйте [font=monospace]Pre[/font] для изменения сообщений, [font=monospace]Post[/font] для наблюдения за ними
[*][b]Отменяйте перехваты в OnPluginEnd[/b] - Всегда очищайте перехваты при выгрузке плагина
[*][b]Проверяйте количество повторяющихся полей[/b] - Проверяйте количество перед доступом к индексам повторяющихся полей
[*][b]Тестируйте с клиентами[/b] - Убедитесь, что сообщения правильно отображаются на стороне клиента
[*][b]Используйте соответствующие типы данных[/b] - Сопоставляйте функции [font=monospace]PbRead[i][/font]/[font=monospace]PbSet[/i][/font] с типами полей
[*][b]Учитывайте производительность[/b] - Избегайте отправки больших сообщений или слишком большого количества сообщений за тик
[/list]

[size=20][b]Часто используемые User Messages[/b][/size]

Вот часто используемые user messages в CS:GO/CS2:

[b]TextMsg[/b] - Отправить текст в чат или консоль:
[list]
[*][font=monospace]text[/font] (string) - Текст сообщения
[*][font=monospace]msg_dst[/font] (int32) - Назначение: 2=консоль, 3=центр, 4=чат
[/list]

[b]HintText[/b] - Показать подсказку на экране:
[list]
[*][font=monospace]text[/font] (string) - Текст подсказки для отображения
[/list]

[b]Fade[/b] - Эффект затемнения экрана:
[list]
[*][font=monospace]duration[/font] (int32) - Длительность затемнения в миллисекундах
[*][font=monospace]hold_time[/font] (int32) - Время удержания в миллисекундах
[*][font=monospace]flags[/font] (int32) - Флаги затемнения (FFADE_IN, FFADE_OUT и т.д.)
[*][font=monospace]clr[/font] (color) - Цвет RGBA
[/list]

[b]ShowMenu[/b] - Отобразить меню:
[list]
[*][font=monospace]validslots[/font] (int32) - Битовая маска допустимых слотов меню
[*][font=monospace]displaytime[/font] (int32) - Как долго отображать (секунды)
[*][font=monospace]needmore[/font] (bool) - Доступны дополнительные страницы
[*][font=monospace]str[/font] (string) - Текст меню
[/list]

[b]Damage[/b] - Показать индикатор урона:
[list]
[*][font=monospace]amount[/font] (int32) - Количество урона
[*][font=monospace]victim[/font] (int32) - Индекс сущности жертвы
[/list]

[b]SayText2[/b] - Форматированное чат-сообщение:
[list]
[*][font=monospace]ent_idx[/font] (int32) - Индекс сущности отправителя
[*][font=monospace]chat[/font] (bool) - Отправить в чат
[*][font=monospace]msg_name[/font] (string) - Строка формата сообщения
[*][font=monospace]params[/font] (repeated string) - Параметры сообщения
[/list]

Обратитесь к [url=https://github.com/SteamDatabase/Protobufs/tree/master/csgo]определениям protobuf CS:GO[/url] для полного списка сообщений и их полей.

[size=20][b]Режимы перехвата[/b][/size]

[list]
[*][b]Pre[/b] ([font=monospace]HookMode.Pre[/font]) - Перехват до обработки сообщения, позволяет модификацию
[*][b]Post[/b] ([font=monospace]HookMode.Post[/font]) - Перехват после обработки сообщения, только чтение для наблюдения
[/list]

Используйте режим Pre, когда хотите изменить или заблокировать сообщения, режим Post для логирования или запуска побочных эффектов.
