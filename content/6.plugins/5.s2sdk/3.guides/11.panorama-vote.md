## Overview

The Panorama Voting System allows you to create custom Yes/No votes that appear in the CS2 UI. Players can vote using the native game interface, making it seamless and familiar.

::callout{type="info"}
**Key Features:**
- Native CS2 UI integration
- Customizable vote messages and pass/fail text
- Selective player targeting using bitmask
- Vote progress callbacks
- Automatic timeout handling
::

## Basic Vote Creation

There are two main functions to start a vote:

1. **`PanoramaSendYesNoVoteToAll`** - Creates a vote for all players
2. **`PanoramaSendYesNoVote`** - Creates a vote for specific players (using recipients bitmask)

### Simple Vote to All Players

::tabs{variant="card"}
::div{label="c#" icon="vscode-icons:file-type-csharp2"}
```csharp
using Plugify;
using static s2sdk.s2sdk;

public unsafe class VoteExample : Plugin
{
    public void OnPluginStart()
    {
        // Create a simple vote for all players
        PanoramaSendYesNoVoteToAll(
            duration: 30.0,                      // 30 seconds to vote
            caller: VOTE_CALLER_SERVER,          // Called by server
            voteTitle: "#SFUI_vote_passed",      // Vote message
            detailStr: "Restart the match?",     // Extra detail
            votePassTitle: "#SFUI_vote_passed",  // Pass message
            detailPassStr: "Match restarting",   // Pass detail
            failReason: 0,                       // No fail reason
            result: OnVoteResult,                // Result callback
            handler: OnVoteComplete              // Completion handler
        );
    }

    private void OnVoteResult(int entity, int voteOption)
    {
        // Called when a player votes
        // voteOption: 0 = No, 1 = Yes
        string vote = voteOption == 1 ? "Yes" : "No";
        PrintToServer($"Player {entity} voted: {vote}\n");
    }

    private void OnVoteComplete(bool passed, int yesVotes, int noVotes)
    {
        // Called when vote finishes
        if (passed)
        {
            PrintToServer($"Vote PASSED! Yes: {yesVotes}, No: {noVotes}\n");
            // Execute action here (e.g., restart match)
        }
        else
        {
            PrintToServer($"Vote FAILED. Yes: {yesVotes}, No: {noVotes}\n");
        }
    }
}
```

## Recipients Bitmask (Selective Voting)

To create a vote for specific players, use `PanoramaSendYesNoVote` with a **recipients bitmask**. Each bit in the `uint64` represents a player slot:

- **Bit 0** (value 1) = Player slot 0
- **Bit 1** (value 2) = Player slot 1
- **Bit 2** (value 4) = Player slot 2
- **Bit N** (value 2^N) = Player slot N

::callout{type="info"}
**Bitmask Construction:** Set bit N to 1 to include player slot N in the vote. The recipients value is the sum of all included player bit values.
::

### Building Recipients Bitmask

Here are examples of building a recipients bitmask for specific players:

::tabs{variant="card"}
::div{label="c#" icon="vscode-icons:file-type-csharp2"}
```csharp
// Include specific players by slot
public ulong BuildRecipients(int[] playerSlots)
{
    ulong recipients = 0;
    foreach (int slot in playerSlots)
    {
        if (slot >= 0 && slot < 64)
        {
            recipients |= (1UL << slot);
        }
    }
    return recipients;
}

// Examples:
// Only player slot 0: recipients = 0b1 = 1
ulong player0Only = 1UL << 0;  // = 1

// Only player slot 5: recipients = 0b100000 = 32
ulong player5Only = 1UL << 5;  // = 32

// Players 0, 1, and 2: recipients = 0b111 = 7
ulong players012 = (1UL << 0) | (1UL << 1) | (1UL << 2);  // = 7

// All players: recipients = 0xFFFFFFFFFFFFFFFF
ulong allPlayers = ulong.MaxValue;  // Use PanoramaSendYesNoVoteToAll instead

// Use in vote
int[] targetPlayers = { 0, 2, 5 };  // Slots 0, 2, 5
ulong recipients = BuildRecipients(targetPlayers);

PanoramaSendYesNoVote(
    duration: 30.0,
    caller: VOTE_CALLER_SERVER,
    voteTitle: "#SFUI_vote_passed",
    detailStr: "Kick player?",
    votePassTitle: "#SFUI_vote_passed",
    detailPassStr: "Player kicked",
    failReason: 0,
    recipients: recipients,  // Only specified players can vote
    result: OnVoteResult,
    handler: OnVoteComplete
);
```

## Vote Helper Functions

The voting system provides several utility functions:

### Check If Vote Is Active

```csharp
bool isVoting = PanoramaIsVoteInProgress();
if (isVoting)
{
    PrintToServer("A vote is currently active!\n");
}
```

### Remove Player From Vote

Removes a player from the current vote pool (they can no longer vote):

```csharp
// Remove player slot 3 from the vote
PanoramaRemovePlayerFromVote(3);
```

### Check If Player Is In Vote Pool

```csharp
int playerSlot = 5;
if (PanoramaIsPlayerInVotePool(playerSlot))
{
    PrintToServer($"Player {playerSlot} can vote\n");
}
else
{
    PrintToServer($"Player {playerSlot} cannot vote\n");
}
```

### Redraw Vote UI for Player

Forces the vote UI to refresh for a specific player:

```csharp
bool success = PanoramaRedrawVoteToClient(playerSlot);
if (success)
{
    PrintToServer($"Vote UI redrawn for player {playerSlot}\n");
}
```

### End Vote Manually

You can manually end an active vote with a specific reason:

```csharp
// End vote with specific reason
PanoramaEndVote(VoteEndReason.VoteFailed);

// Available reasons:
// - VoteEndReason.VotePassed
// - VoteEndReason.VoteFailed
// - VoteEndReason.VoteCancelled
```

Only allow Counter-Terrorists to vote on changing the map:

```csharp
public void StartMapVote()
{
    // Build recipients bitmask for CT team only
    ulong recipients = 0;
    for (int slot = 0; slot < 64; slot++)
    {
        int handle = PlayerSlotToEntHandle(slot);
        if (handle == -1) continue;

            PrintToServer($"Player {entity} voted: {voteOption}\n");

            if (passed)
```

### Example 2: Kick Vote Started by Player

A player initiates a vote to kick another player:

```csharp
public void StartKickVote(int callerSlot, int targetSlot)
{
    // Prevent voting if already active
    if (PanoramaIsVoteInProgress())
    {
        PrintToChat(callerSlot, "A vote is already in progress!");
        return;
    }

    // Get target player name
    int targetHandle = PlayerSlotToEntHandle(targetSlot);
    string targetName = GetPlayerName(targetHandle);

    // All players can vote (except the target)
    PanoramaSendYesNoVoteToAll(
        duration: 30.0,
        caller: callerSlot,  // Vote initiated by player
        voteTitle: "#SFUI_vote_kick_player",
        detailStr: targetName,
        votePassTitle: "#SFUI_vote_passed",
        detailPassStr: $"Kicking {targetName}",
        failReason: 0,
        result: (entity, voteOption) =>
        {
            // Log votes
            PrintToServer($"Player {entity} voted {voteOption} on kick\n");
        },
        handler: (passed, yesVotes, noVotes) =>
        {
            if (passed)
            {
                // Remove target from vote first
                PanoramaRemovePlayerFromVote(targetSlot);

                // Kick the player
                ServerCommand($"kickid {targetSlot}");
                PrintToChatAll($"{targetName} was kicked by vote!");
            }
            else
            {
                PrintToChatAll($"Vote to kick {targetName} failed.");
            }
        }
    );

    // Remove target player from vote pool
    PanoramaRemovePlayerFromVote(targetSlot);

    PrintToChatAll($"{GetPlayerName(PlayerSlotToEntHandle(callerSlot))} started a vote to kick {targetName}");
}
```

## Translation Strings

The voting system uses CS2's built-in translation strings. Common ones include:

| Translation String | Description |
|--------------------|-------------|
| `#SFUI_vote_passed` | Vote passed message |
| `#SFUI_vote_failed` | Vote failed message |
| `#SFUI_vote_kick_player` | Kick player vote |
| `#Panorama_vote` | Generic vote message |

::callout{type="warning"}
**Translation String Requirement:** Only use translation strings that start with `#SFUI_vote` or `#Panorama_vote`. Other strings may not display correctly in the voting UI.
::

## Method Reference

### Starting Votes

| Method | Description |
|--------|-------------|
| `PanoramaSendYesNoVoteToAll(duration, caller, voteTitle, detailStr, votePassTitle, detailPassStr, failReason, result, handler)` | Start a vote for all players |
| `PanoramaSendYesNoVote(duration, caller, voteTitle, detailStr, votePassTitle, detailPassStr, failReason, recipients, result, handler)` | Start a vote for specific players (using bitmask) |

#### Parameters

- **`duration`** (double) - Maximum time in seconds for the vote
- **`caller`** (int) - Player slot who initiated the vote, or `VOTE_CALLER_SERVER` for server votes
- **`voteTitle`** (string) - Translation string for the vote message (e.g., `#SFUI_vote_passed`)
- **`detailStr`** (string) - Additional detail text for the vote
- **`votePassTitle`** (string) - Translation string shown when vote passes
- **`detailPassStr`** (string) - Additional detail text when vote passes
- **`failReason`** (int) - Reason code for vote failure
- **`recipients`** (uint64) - Bitmask of player slots who can vote (only for `PanoramaSendYesNoVote`)
- **`result`** (callback) - Function called when a player votes: `(int entity, int voteOption) => {}`
  - `entity`: Entity handle of voting player
  - `voteOption`: 0 = No, 1 = Yes
- **`handler`** (callback) - Function called when vote completes: `(bool passed, int yesVotes, int noVotes) => {}`
  - `passed`: Whether the vote passed
  - `yesVotes`: Number of yes votes
  - `noVotes`: Number of no votes

### Vote Management

| Method | Description |
|--------|-------------|
| `PanoramaIsVoteInProgress()` | Returns `true` if a vote is currently active |
| `PanoramaEndVote(VoteEndReason reason)` | Manually end the current vote with a reason |

### Player Management

| Method | Description |
|--------|-------------|
| `PanoramaRemovePlayerFromVote(int playerSlot)` | Remove a player from the current vote pool |
| `PanoramaIsPlayerInVotePool(int playerSlot)` | Check if a player can vote in the current vote |
| `PanoramaRedrawVoteToClient(int playerSlot)` | Force refresh the vote UI for a player |

## Tips and Best Practices

1. **Check for active votes** - Always use `PanoramaIsVoteInProgress()` before starting a new vote
2. **Handle disconnections** - Remove disconnected players from votes using `PanoramaRemovePlayerFromVote()`
3. **Use appropriate durations** - 20-30 seconds is typical for most votes
4. **Validate recipients** - Ensure at least one player is in the recipients bitmask
5. **Use translation strings** - Stick to `#SFUI_vote` or `#Panorama_vote` strings for proper localization
6. **Log vote results** - Track voting patterns for debugging and analytics
7. **Prevent vote spam** - Implement cooldowns between votes
8. **Remove vote targets** - Don't allow players being voted on to participate
9. **Handle edge cases** - Consider what happens if all voters disconnect
